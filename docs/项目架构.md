# HumanOS 项目架构（对照 PRD）

**约定**：仓库内 **仅使用 JavaScript（.js）**，不引入 TypeScript。前后端 **物理分离、职责分离**，各自 **目录内模块化**，通过 **WebSocket 信令协议** 与后续 **WebRTC P2P** 协作。

## 2. 后端：`backend/signal-server`（模块化）

| 模块路径 | 职责 |
|----------|------|
| `src/config.js` | 端口、路径等环境配置 |
| `src/http/healthServer.js` | 轻量 HTTP（如 `/health`） |
| `src/registry/RoomRegistry.js` | 控制码 ↔ 房间 ↔ 连接（Agent / Controller） |
| `src/ws/messageRouter.js` | 解析 JSON、校验、分发到 registry |
| `src/ws/attachWebSocket.js` | 将 `ws` 连接挂到 router、处理 close |
| `src/protocol/messageTypes.js` | 消息 `type` 常量，避免魔法字符串 |
| `src/index.js` | 组装 HTTP + WebSocket 并启动 |

**扩展方式**：新增信令类型时，在 `protocol` 增加常量，在 `messageRouter` / `RoomRegistry` 增加分支或策略对象，避免单文件无限膨胀。

## 3. 前端：`frontend/desktop`（Vue 3 + 模块化）

### 3.1 进程与构建

| 部分 | 路径 | 说明 |
|------|------|------|
| 主进程 | `electron-main/` | 窗口加载；`inputDispatcher.js`：`move` / `click` / `wheel`（`@nut-tree-fork/nut-js`，需系统权限） |
| 预加载 | `preload/` | `contextBridge`：`getDefaultSignalUrl`、`inputDispatch`（转发主进程注入） |
| 渲染层 | `src/`、`index.html` | **Vite**；**Vue 3 SFC（纯 JS）**；**Tailwind**；**lucide-vue-next** |

### 3.2 渲染层代码组织

| 路径 | 职责 |
|------|------|
| `src/App.vue` | 主界面与 AI 侧栏 |
| `src/composables/useRemoteSession.js` | 信令 + WebRTC 会话编排（控制码、Agent/Controller、断开） |
| `src/lib/webrtc/` | `rtcConfig.js`、`controlChannel.js`、`agentRtcSession.js`、`controllerRtcSession.js` |
| `src/lib/signal/` | WebSocket 信令客户端 |
| `src/lib/config/` | 信令 URL |
| `src/lib/codeGenerator.js` | 控制码生成与展示格式 |

原 `renderer/` 纯脚本入口已移除，由 Vite + Vue 替代。

### 3.3 桌面本地开发

```bash
npm run dev -w @humanos/desktop
```

并行启动 Vite（5173）与 Electron（见 workspace `package.json`）。

### 3.4 开发态「被控 + 控制」双客户端自测（同一台电脑）

**不要**开两个 `npm run dev:desktop`：会起 **两个 Vite**，5173 端口冲突（`strictPort`）。

推荐 **1 个 Vite + 2 个 Electron**：

```bash
# 终端 1 — 信令
npm run dev:signal

# 终端 2 — 仅 Vite（一直开着）
npm run dev:desktop:vite

# 终端 3 — 第一个 Electron（会等 Vite 就绪）
npm run dev:desktop:electron

# 终端 4 — 第二个 Electron（独立 userData，避免与窗口 3 抢配置）
npm run dev:desktop:electron:ctrl
```

若 Vite 已在跑，第 3 个及以后的窗口也可改用（不等 wait-on）：

```bash
npm run dev:desktop:electron:open
npm run dev:desktop:electron:ctrl
```

流程：窗口 A 选 **被控** → 启动服务；窗口 B 选 **控制** → 输入同一控制码 → 建立连接。`dev:electron:ctrl` 会设置 `HUMANOS_USER_DATA_SUFFIX`，主进程将 `userData` 指到独立目录（见 `electron-main/main.js`）。

跨机器时，控制端须把信令地址配成 **信令所在机器的局域网 IP**（默认 `127.0.0.1` 在对方电脑上不是本机）。

## 4. 前后端通信（信令）

- **传输**：`ws://<host>:<port>/ws`（生产可换 WSS + 反向代理）
- **载荷**：JSON，`type` 字段区分行为（见 `backend/.../protocol/messageTypes.js` 与后端实现）
- **原则**：仅会话控制与 SDP/ICE；**音视频与控制数据走 WebRTC**（`src/lib/webrtc` + `useRemoteSession`）

---

## 5. 本地开发命令

```bash
npm install
npm run dev:signal           # 信令
npm run dev:desktop:vite     # 仅 Vite（单实例）
npm run dev:desktop:electron # Electron #1
npm run dev:desktop:electron:ctrl # Electron #2（控制端）
```

单机「一键」仍可用：`npm run dev:desktop`（单窗口）。双客户端自测见 **§3.4**。

## 7. 技术约束（本仓库）

- **不使用 TypeScript**：无 `tsconfig`、无 `.ts` 源文件。
- **不将信令逻辑塞进 Electron 主进程**：信令服务独立进程，便于部署与伸缩。
- **模块化**：单文件职责单一；新增功能优先新模块或子目录，而非拉长 `index.js`。
- **模块格式**：信令服务与 Electron **主进程**为 **CommonJS**；**Vue 渲染层**由 **Vite** 打包，源码为 **纯 JS + Vue SFC**，**无 TypeScript**。

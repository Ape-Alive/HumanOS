# HumanOS 项目架构（对照 PRD）

**约定**：仓库内 **仅使用 JavaScript（.js）**，不引入 TypeScript。前后端 **物理分离、职责分离**，各自 **目录内模块化**，通过 **WebSocket 信令协议** 与后续 **WebRTC P2P** 协作。

## 2. 后端：`backend/signal-server`（模块化）

| 模块路径 | 职责 |
|----------|------|
| `src/config.js` | 端口、路径等环境配置 |
| `src/http/healthServer.js` | 轻量 HTTP（如 `/health`） |
| `src/registry/RoomRegistry.js` | 控制码 ↔ 房间 ↔ 连接（Agent / Controller） |
| `src/ws/messageRouter.js` | 解析 JSON、校验、分发到 registry |
| `src/ws/attachWebSocket.js` | 将 `ws` 连接挂到 router、处理 close |
| `src/protocol/messageTypes.js` | 消息 `type` 常量，避免魔法字符串 |
| `src/index.js` | 组装 HTTP + WebSocket 并启动 |

**扩展方式**：新增信令类型时，在 `protocol` 增加常量，在 `messageRouter` / `RoomRegistry` 增加分支或策略对象，避免单文件无限膨胀。

## 3. 前端：`frontend/desktop`（Vue 3 + 模块化）

### 3.1 进程与构建

| 部分 | 路径 | 说明 |
|------|------|------|
| 主进程 | `electron-main/` | 窗口；`inputDispatcher.js`：`move` / `click` / `wheel` / `key` / `text`（`@nut-tree-fork/nut-js`）；`agentDb/`：任务与日志持久化（sql.js） |
| 预加载 | `preload/` | `contextBridge`：信令/剪贴板/键鼠、`agentDb`、`saveMarkdownReport` |
| 渲染层 | `src/`、`index.html` | **Vite**；**Vue 3 SFC（纯 JS）**；**Tailwind**；**lucide-vue-next** |

### 3.2 渲染层代码组织

| 路径 | 职责 |
|------|------|
| `src/App.vue` | 主界面、远程会话 UI、AI Copilot 侧栏与任务编排 |
| `src/composables/useRemoteSession.js` | 信令 + WebRTC；`remoteControlReady`（DataChannel open 响应式）、`sendRemoteControl` / `isRemoteControlReady` 供 AI 与 UI |
| `src/composables/useAiAgentOrchestrator.js` | AI 多轮截图–规划–执行–断言–写库 |
| `src/lib/ai/**` | Agent / Plugin / Prompt / 报告 |
| `src/lib/webrtc/` | `rtcConfig.js`、`controlChannel.js`、`agentRtcSession.js`、`controllerRtcSession.js` |
| `src/lib/signal/` | WebSocket 信令客户端 |
| `src/lib/config/` | 信令 URL |
| `src/lib/codeGenerator.js` | 控制码生成与展示格式 |

原 `renderer/` 纯脚本入口已移除，由 Vite + Vue 替代。

### 3.3 桌面本地开发

```bash
npm run dev -w @humanos/desktop
```

并行启动 Vite（5173）与 Electron（见 workspace `package.json`）。

### 3.4 开发态「被控 + 控制」双客户端自测（同一台电脑）

**不要**开两个 `npm run dev:desktop`：会起 **两个 Vite**，5173 端口冲突（`strictPort`）。

推荐 **1 个 Vite + 2 个 Electron**：

```bash
# 终端 1 — 信令
npm run dev:signal

# 终端 2 — 仅 Vite（一直开着）
npm run dev:desktop:vite

# 终端 3 — 第一个 Electron（会等 Vite 就绪）
npm run dev:desktop:electron

# 终端 4 — 第二个 Electron（独立 userData，避免与窗口 3 抢配置）
npm run dev:desktop:electron:ctrl
```

若 Vite 已在跑，第 3 个及以后的窗口也可改用（不等 wait-on）：

```bash
npm run dev:desktop:electron:open
npm run dev:desktop:electron:ctrl
```

流程：窗口 A 选 **被控** → 启动服务；窗口 B 选 **控制** → 输入同一控制码 → 建立连接。`dev:electron:ctrl` 会设置 `HUMANOS_USER_DATA_SUFFIX`，主进程将 `userData` 指到独立目录（见 `electron-main/main.js`）。

跨机器时，控制端须把信令地址配成 **信令所在机器的局域网 IP**（默认 `127.0.0.1` 在对方电脑上不是本机）。

## 4. 前后端通信（信令）

- **传输**：`ws://<host>:<port>/ws`（生产可换 WSS + 反向代理）
- **载荷**：JSON，`type` 字段区分行为（见 `backend/.../protocol/messageTypes.js` 与后端实现）
- **原则**：仅会话控制与 SDP/ICE；**音视频与控制数据走 WebRTC**（`src/lib/webrtc` + `useRemoteSession`）

## 7. AI 视觉任务自动化（Agent / Data / Plugin）

控制端在 **WebRTC 会话已连接**、**DataChannel 控制通道就绪**、**远程 `<video>` 有画面** 时，侧栏可发起「任务 → 截图 → 规划 → 执行 → 验收」循环；模型需支持 **OpenAI 兼容的多模态**（`/v1/chat/completions` + `image_url`），例如 `gpt-4o`、`gpt-4o-mini` 等。

### 7.1 闭环数据流

```
任务描述（自然语言）
    ↓
截取远程 video 帧（JPEG，归一化坐标基准）
    ↓
Vision：理解当前界面（中文描述）
    ↓
Planner：输出 JSON 步骤（click / move / wheel / type_text / wait_ms）
    ↓
Action：映射为 DataChannel 控制指令 → 被控端 nut-js
    ↓
再次截图 + **视觉验收（截图）**：未通过则进入下一轮规划；通过则结束
    ↓
（循环直至 maxRounds；若仍未通过则做**最终截图验收**）
    ↓
生成 Markdown 测试报告 + 写入 sql.js 数据库
```

### 7.2 Agent Layer（`src/lib/ai/`，渲染进程，可替换）

| 模块 | 路径 | 职责 |
|------|------|------|
| Vision Module | `agent/visionModule.js` | 结合 `promptManager` 与 Vision Provider，对截图做界面理解 |
| Planner Module | `agent/plannerModule.js` | 根据目标 + Vision 文本生成 **仅 JSON** 的步骤规划 |
| Action Generator | `agent/actionGenerator.js` | 解析 JSON；`nx,ny∈[0,1000]` → 远端视频像素；生成 `move`/`click`/`wheel`/`text` |
| Assertion Engine | `agent/assertionEngine.js` | `runAssertionWithVision`：结合截图输出 `{ passed, evidence }`；`runAssertion` 为纯文本备用 |
| Prompt Manager | `prompts/promptManager.js` | 各阶段 system 提示词集中管理 |

编排入口：`src/composables/useAiAgentOrchestrator.js`（`createAiAgentRunner`），由 `App.vue` 在 `onMounted` 时绑定 `addLog`、`activeAiProfile`、`remoteVideoRef`、`rs.sendRemoteControl` 等。

### 7.3 Data Layer（主进程 + sql.js）

使用 **[sql.js](https://sql.js.org/)**（SQLite 的 WASM 实现），数据库文件位于 **`app.getPath('userData')/humanos-agent.sqlite`**，随写随 `export` 持久化，无需原生 `better-sqlite3` 编译。

| 逻辑表 | 用途 |
|--------|------|
| `tasks` | 任务目标、状态、关联配置名 |
| `execution_logs` | 每轮执行日志与结构化 `payload_json` |
| `screenshots` | 关键帧 JPEG 二进制（控制端截图入库） |
| `test_results` | 每任务一条 Markdown 报告 + `summary_json` |

实现：`electron-main/agentDb/repository.js`；IPC 注册：`electron-main/agentDb/ipc.js`；预加载暴露 `window.humanos.agentDb.*` 与 `window.humanos.saveMarkdownReport`。

### 7.4 Plugin Layer（`src/lib/ai/plugins/`）

| 适配器 | 说明 |
|--------|------|
| `openAiVisionChat.js` | `createOpenAiCompatibleAdapter`：`/chat/completions` + 可选 `image_url` |
| `createAiProviderAdapter` | 预留 `gemini` / `claude` 分支（当前抛错，提示扩展点） |
| `visionProviderAdapter.js` | 将同一适配器包装为「Vision 专用」接口，便于替换为 Gemini Vision 等 |

### 7.5 测试报告

任务结束后生成 Markdown（`report/buildMarkdownReport.js`），侧栏 **「导出 Markdown 测试报告」** 调用主进程 `dialog.showSaveDialog` 写入用户选定路径。

## 8. 技术约束（本仓库）

- **不使用 TypeScript**：无 `tsconfig`、无 `.ts` 源文件。
- **不将信令逻辑塞进 Electron 主进程**：信令服务独立进程，便于部署与伸缩。
- **模块化**：单文件职责单一；新增功能优先新模块或子目录，而非拉长 `index.js`。
- **模块格式**：信令服务与 Electron **主进程**为 **CommonJS**；**Vue 渲染层**由 **Vite** 打包，源码为 **纯 JS + Vue SFC**，**无 TypeScript**。
